# Проект motion

## Анимации и токены
- Используй только easing и длительности из библиотеки токенов (`tokens.css` / массивы `EASING_TOKENS`, `DURATION_TOKENS`). Никогда не вводи произвольные cubic-bezier или значения в ms, если пользователь явно не попросил отдельно.
- При просьбе «сделать плавнее / медленнее / быстрее» меняй только длительность на соседний токен (`fast` 240 / `base` 340 / `slow` 540), не трогая easing, если не сказано иное.

## Сборка и шифрование
- Источник правды — `source.html` (незашифрованный, открывается локально без пароля). В Git/GitHub не попадает (в `.gitignore`).
- Все правки контента делай в `source.html`, затем запускай `./build.sh` — он шифрует `source.html` в `index.html` через Staticrypt v3 (пароль из `.staticrypt-password`, соль из `.staticrypt.json`).
- В Git коммить только `index.html` (зашифрованный) — на GitHub проект остаётся под паролем. `source.html` и `.staticrypt-password` остаются только локально.

## Dev-контролы для настройки анимаций
- Когда пользователь просит «включи настройки анимации» — вставь блок из `dev/anim-controls.html` (мастер-копия) в `source.html` перед `</body>`. Это два ряда чипов (duration × easing), которые перебирают сочетания на текущей выбранной анимации.
- «Выключи настройки» — удали из `source.html` блок от `<!-- DEV:ANIM-CONTROLS:START -->` до `<!-- DEV:ANIM-CONTROLS:END -->`. Мастер-копия в `dev/` остаётся для повторного включения.
- Контролы видны только локально в `source.html`. `build.sh` вырезает блок по маркерам, поэтому в `index.html` (и на GitHub) их не будет.
- Когда пользователь выбрал понравившееся сочетание — впиши его в `defaults` нужной анимации в `source.html`, затем `./build.sh`.
